## 🔧 Troubleshooting & Common Issues

### The overlay does not appear after launching

Make sure you are running the environment with administrator or root privileges. On Windows, right-click and select "Run as administrator." On macOS, ensure accessibility permissions are granted.

### The auto dress module selects incorrect items

This usually means the local theme cache is outdated. Switch the `theme_database` parameter to `cloud` or `hybrid` in your configuration file and restart.

```bash
# Force a theme cache refresh
cd ~/dress-impress-toolkit/
rm -rf cache/themes/*
./start.sh --refresh-themes
```

### High memory usage during long sessions

The logging system accumulates data over time. Set `logging_level` to `warn` or `error` to reduce memory consumption, and restart every two to three hours during marathon sessions.

### Android overlay permission denied

Navigate to **Settings → Apps → Special Access → Display Over Other Apps** and manually enable the permission for your runtime application.

### iOS certificate trust error

Go to **Settings → General → VPN & Device Management**, find the developer certificate, and tap **Trust**. You may need to reconnect to the internet for verification.

### Connection timeout when syncing the theme database

Check your internet connection first. If the issue persists, try switching to a different DNS provider (such as `1.1.1.1` or `8.8.8.8`) or enable the VPN recommended in the safety tips above.
